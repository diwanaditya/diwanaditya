Projects

System Inspector

Python Async I/O psutil Tkinter OS APIs Data processing
Abstract. System Inspector is a real-time system monitoring application designed to unify CPU, memory, disk, and network analysis within a single interactive dashboard. The tool collects metrics at sub-second intervals using asynchronous I/O and the psutil library, enabling accurate, low-overhead observation of system state without perturbing the processes being monitored. It implements a health scoring engine that assigns numerical reliability scores to system components based on threshold-aware anomaly detection, allowing users to quickly identify degraded or overloaded resources. A timeline replay module stores historical snapshots in a structured in-memory buffer, enabling post-hoc inspection and root-cause analysis for performance incidents. The dashboard is built with Tkinter and follows a modular component architecture, where each monitoring subsystem is independently extensible. Integrated PDF reporting generates structured system summaries on demand, suitable for documentation and incident reporting.

NetScope

Python REST APIs JSON CLI Networking
Abstract. NetScope is a command-line intelligence tool for collecting structured information on ASNs, IP addresses, and ISPs through multiple public APIs. The tool implements a unified request pipeline with configurable retry logic and provider fallback mechanisms, ensuring reliable data retrieval even when individual API endpoints are unavailable or rate-limited. At its core is an extensible data parser framework that normalizes inconsistent response schemas across data sources into a common internal representation, significantly reducing the complexity of downstream processing. Output is rendered in clean, formatted tables directly in the terminal, optimized for rapid network assessment and investigative workflows. NetScope is designed to be composable, allowing its output to feed into shell pipelines or be extended with additional data providers through a well-defined plugin interface.

Research
Ongoing  ·  2024 – Present

LLM Reliability and Hallucination Detection

This research investigates the conditions under which large language models generate incorrect, unsafe, or factually drifted outputs — a problem commonly referred to as hallucination. The work proceeds along four parallel tracks. First, semantic entropy and complementary uncertainty estimation are applied via LLM APIs to detect when a model's output distribution suggests unreliable generation. Second, adversarial stress testing pipelines systematically probe model failure modes by constructing challenging and diverse prompt conditions designed to surface edge-case behaviors. Third, a structured risk scoring framework ranks model outputs along axes of reliability, safety impact, and factual drift — providing a quantitative basis for model comparison and triage. Fourth, domain-specific evaluation sets spanning high-risk categories — including jailbreak probes and legal compliance queries — are used to measure model robustness under realistic adversarial conditions. The technical stack includes PyTorch, Hugging Face Transformers, Scikit-learn, spaCy, and NLTK.
